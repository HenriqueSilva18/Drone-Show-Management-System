# Glossary of Terms for the Shodrone System

| Term | Definition |
|------|------------|
| ANTLR | Tool that must be used to implement support for the domain-specific language (DSL) and validation of drone languages. |
| Admin | Role responsible for user management in the system. |
| Backoffice | Internal administrative system used by Shodrone to manage customers, figures, shows, and operations. |
| Collision | When two drones occupy the same space at the same time during a show or simulation. |
| CRM Collaborator | Shodrone employee responsible for customer service, registering show requests, and managing customer representatives. |
| CRM Manager | Role responsible for customer management and show request assignment to designers. |
| Customer | European business or public entity that contracts Shodrone's services. Has a European VAT number, address, and representatives. |
| Customer Representative | Person who represents a customer and interacts with Shodrone. Has email, name, and position. |
| Domain Model | Conceptual model created using Domain-Driven Design principles to represent the system's entities and relationships. |
| Domain-Specific Language (DSL) | High-level language used to describe drone figures and shows, allowing vendor independence. |
| Drone | Device that performs aerial maneuvers in a show. Each drone has a model and serial number in the inventory. |
| Drone Tech | Role responsible for drone-related technical operations, including simulation and code generation. |
| Drone Type | Classification of drones based on capabilities and features, used when implementing elements of a figure. |
| Dynamic Figure | A figure where drones move in the air performing a routine with a fixed duration. |
| Elemental Volume | The smallest cubic unit in the 3D simulation space used to detect potential collisions. |
| Exclusivity | Option for customers to have exclusive rights to a specific figure for a defined period. |
| Figure | Composition of drones arranged to create a specific formation or pattern in the sky. Can be static or dynamic. |
| Figure Catalogue | Collection of all figures available for use in shows. |
| Orchestrator | Central server that coordinates drones during a show or simulation, giving orders to initiate figures and receiving completion feedback. |
| Plugin | Module that can be added to the system to extend its functionality, such as for DSL processing or document validation. |
| Proposal | Document sent to customers detailing the show's configuration, including figures, drones, and preview video. |
| Show | A sequence of figures performed by drones with a specific duration and location. |
| Show Designer | Role responsible for creating and updating figures based on customer requests. |
| Show Request | Initial request from a customer for a drone show, including location, time, requested number of drones, and duration. |
| Shodrone | Company providing customized drone multimedia shows, the client for this software project. |
| Simulation | Process of running a figure or show in a virtual environment to test for potential collisions or other issues. |
| Sprint | Time-boxed period in the Scrum methodology during which specific work must be completed and ready for review. |
| Static Figure | A figure where drones maintain their position in the air without movement. |
| Template | Predefined format for documents like proposals that can be configured by CRM Managers. |
| User | Person with access to the system, identified by a unique email from Shodrone's domain. |
| User Story | Description of software functionality from the perspective of the end user. |
| VIP Customer | Customer with priority status in the system, receiving preferential treatment over Regular customers. |
